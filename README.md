# Machine Learning for X-ray Diffractometry: Why Physical Grounding is Essential

This repository contains the code accompanying the paper **"Machine Learning for X-ray Diffractometry: Why Physical Grounding is Essential"** by Redad Mehdi, Weiqi Yue, Ozan Dernek, Matthew A. Willard, Roger H. French, and Frank Ernst (Case Western Reserve University).

The central finding of the paper is that general-purpose image recognition pre-training (ImageNet) and physics-driven synthetic pre-training can yield nearly identical accuracy on XRD phase fraction prediction, while differing fundamentally in *how* they arrive at those predictions. Robustness testing reveals that the ImageNet-initialized model exploits surface texture and background statistics rather than crystallographic features, whereas the synthetically pre-trained model correctly roots its predictions in Bragg peak evidence. This repository provides the tools to reproduce those results.

## Overview

The workflow proceeds in three stages:

**1. Synthetic pre-training.** The MiT-B2 encoder is pre-trained on a large corpus of simulated 2D XRD patterns generated by a kinematic diffraction simulator (see [Related Repositories](#related-repositories)). Curriculum learning is applied: the model trains first on single-phase patterns, then two-phase mixtures, then three-phase mixtures. Synthetic patterns encode phase fractions directly in their filenames following the convention `Mix-A{alpha}B{beta}G{gamma}-T{temp}K-Bkg-{id}.npy`, e.g. `Mix-A0.40B0.50G0.10-T400-Bkg-19.npy`.

**2. Fine-tuning on experimental data.** Both the synthetically pre-trained model and an ImageNet-initialized baseline are fine-tuned on labeled experimental diffractograms from Ti-6Al-4V synchrotron experiments, with as few as 10–250 patterns.

**3. Robustness evaluation.** Two diagnostic tests expose differences in model behavior that accuracy metrics alone cannot reveal. These are implemented as interactive demos in the `notebooks/` directory.

The material system is Ti-6Al-4V (Ti64), which contains three crystallographic phases: alpha-Ti (HCP), beta-Ti (BCC), and gamma-Fe (FCC, from the steel sample container). Experimental data come from two independent synchrotron experiments at the Cornell High Energy Synchrotron Source (CHESS).

## Installation

Python 3.10 or later is recommended. To install the dependencies:

```bash
git clone https://github.com/cwru-sdle/causal-ml-xrd.git
cd causal-ml-xrd
pip install -r requirements.txt
```

Training was performed on an NVIDIA A100 GPU (80 GB VRAM). The MiT-B2 model uses a batch size of 1 with gradient accumulation over 16 steps. A GPU with at least 16 GB of VRAM is recommended for replication.

## Data

**Synthetic data.** A sample of synthetic multi-phase diffractograms is included in `data/synthetic/`. The full synthetic corpus can be generated using the kinematic diffraction simulator (see [Related Repositories](#related-repositories)). Patterns are simulated for alpha-Ti, beta-Ti, and gamma-Fe at temperatures from 300 K to 1300 K in 100 K steps, with random variation in grain size, peak shape parameters (pseudo-Voigt mixing), shot noise, and Chebyshev background coefficients. Multi-phase patterns are produced by incoherent superposition of single-phase patterns weighted by unit-cell-number fractions.

**Experimental data.** Sample experimental diffractograms and their phase-fraction labels are included for both datasets:

- `data/dataset1/` — Collected at the FAST beamline at CHESS at 67.31 keV (Bonnie et al.). Patterns are stored as `.tif` files and were averaged over 10-frame groups, dark-current corrected, and normalized by global min-max scaling. Labels are in `labels.json`.
- `data/dataset2/` — Collected at 69.53 keV with variable detector distance (Strantza et al.). Patterns are stored as `.npy` files, normalized by global min-max scaling. Labels are in `labels.json`.

The full experimental datasets are not included due to size constraints. Please refer to the original publications for data access.

## Robustness Evaluation

The two robustness tests from the paper are implemented as interactive Jupyter notebooks in `notebooks/`:

**`Stress-Tests.ipynb`** runs the geometric stability (zoom) test and the physical causality (radial masking) test, and generates Grad-CAM visualizations for both the ImageNet-initialized and synthetically pre-trained models. The zoom test progressively magnifies the diffractogram (1.0x, 1.5x, 2.0x); the masking test applies centered radial masks of increasing radius (300, 600, 800 pixels), progressively occluding the Bragg peaks. A physically grounded model should shift its predictions as evidence is removed; a texture-biased model will not.

**`Weight-Space.ipynb`** computes the pairwise Euclidean distances between encoder parameter vectors across all model variants (ImageNet initialization, synthetic initialization, and each fine-tuned variant for Datasets 1 and 2). This analysis shows that both models converge to similar MAE via fundamentally different solutions in weight space.
